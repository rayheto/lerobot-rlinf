# 代码检索规则（强制）

- 任何代码探索 / 查找 / 追踪 / "在哪里"/"怎么实现"/"调用关系" 类问题，**必须先调用 `codegraph_explore`**，不要直接用 Grep/Glob/Read 去探索。
- `codegraph_explore` 既能跑自然语言问题，也能传一组符号/文件名，返回的是按文件分组的源代码，已经是 Read-equivalent。绝大多数情况下一次调用就够。
- 仅当满足以下任一情况时，才允许 fallback 到 Grep/Glob/Read：
  1. `codegraph_explore` 返回为空或明显不覆盖问题；
  2. 需要确认 codegraph 没有返回的某个具体细节（字符串字面量、配置值等）；
  3. 目标文件路径已知，直接 Read 即可。
- 不要"先 grep 一下再说"。违反此规则视为错误。
